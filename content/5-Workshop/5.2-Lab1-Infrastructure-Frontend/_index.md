Welcome to **Lab 1** of our workshop. In this module, we will lay the foundation for our system architecture on AWS and deploy the frontend application.

This is a crucial first step to ensure that our application runs in a secure, isolated, and tightly controlled network environment.

## Lab 1 Architecture

In this module, we will focus on the following key components:
1. **Network**: Building a Virtual Private Cloud (VPC) complete with Public and Private Subnets.
2. **Security**: Enforcing secure access using IAM Roles and controlling network traffic via Security Groups.
3. **Frontend Hosting**: Deploying the web application (Frontend) on AWS so it becomes publicly accessible via the Internet.

## Step-by-Step Instructions

Lab 1 is broken down into the following smaller steps. Please follow them in order:

- **[1. Configure VPC](1-vpc/)**: Set up an isolated network environment (VPC, Subnets, Internet Gateways, NAT Gateways).
- **[2. Configure Security](2-security/)**: Define firewall rules (Security Groups) and IAM permissions for our services.
- **[3. Deploy Frontend](3-deploy-frontend/)**: Upload and host the frontend source code to make the website live.
